# world/

Unkĺike the proposed usage of the `world/` directory, I have created the whole
game structure under this folder like `typeclasses/`, `objects/`, `rooms`,
`scripts/`, `utils/` and `prototypes/`.

So the whole game structure so far is under `world/`, except for main commands,
which are still placed under `commands/`.

## Changes to the world/ directory

Users **can** make changes to the `world/` directory if they found a bug or have
a really good idea which contains changes to the typeclasses for example. But
these have to be made carefully and **not without discussion**! Best to open an
issue first before creating and sending a Pull Request.
